# S52 — Chat session tabs with auto-naming

## User story

As a **runner**,
I want **my chat sessions to be auto-named and shown as tabs**
so that **I can quickly switch between conversations and keep history visible**.

## Acceptance criteria

- [ ] Sessions displayed as horizontal tabs above the chat area
- [ ] Each tab shows an auto-generated name derived from the first user message
- [ ] Auto-name is generated by taking the first ~50 chars of the first user message, truncated at a word boundary with ellipsis
- [ ] Session title stored in the `chat_sessions` table (new `title` column)
- [ ] Backend command `rename_chat_session` for manual rename support
- [ ] Clicking a tab loads that session's messages (history stays open)
- [ ] Each tab has a close (X) button that deletes the session
- [ ] New Chat (+) button creates a new tab and switches to it
- [ ] Active tab visually distinct from inactive tabs
- [ ] Tabs overflow handled with horizontal scrolling
- [ ] Chat history sidebar removed (replaced by tabs)
- [ ] Empty state when no sessions exist prompts user to start chatting

## Technical notes

### Backend changes (Rust)

1. **Database migration**: Add `title TEXT` column to `chat_sessions` table (nullable, auto-populated).
2. **`SessionData` model**: Add `title: Option<String>` field.
3. **New command `rename_chat_session`**: Updates session title.
4. **Auto-title in `send_message`**: After the first user message in a new session, set the title based on the message content.
5. **Storage layer**: Add `update_chat_session_title(&self, session_id, title)` method, update `create_chat_session` and `get_chat_sessions` to include title.

### Frontend changes (TypeScript)

1. **`Chat.tsx`**: Replace sidebar history panel with a tab bar component.
2. **Tab bar**: Horizontal, scrollable, positioned between the chat toolbar and message area.
3. **Session interface**: Add `title?: string` field.
4. **Tab close**: X button with `stopPropagation` to avoid switching on close.
5. **Auto-scroll tabs**: When a new tab is created, scroll it into view.

### Auto-naming strategy

- Take first user message content
- Truncate to ~50 characters at nearest word boundary
- Append ellipsis if truncated
- If no user message yet, show "New Chat"
- Title is set server-side in `send_message` when first message is sent to a new (untitled) session

## Tests required

- Rust unit: `update_chat_session_title` CRUD, auto-title on first message
- Rust unit: verify `SessionData` serialization with title field
- TypeScript: `npx tsc --noEmit` passes
- Integration: create session -> send message -> verify title auto-set
- Edge cases: very long first message truncation, empty session title, special characters

## Out of scope

- Manual rename via double-click (future enhancement)
- Drag-and-drop tab reordering
- Tab pinning
- Session search/filtering
